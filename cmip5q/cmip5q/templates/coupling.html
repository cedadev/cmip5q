There is one "coupling" entry for each input requirement you declared in your model component description. It may be you need to go back to the model description to add input requirements, so you can describe them here.

Recall there are three types of inputs, and the nature of the what we ask depends on what sort of input is required.

  • Boundary Conditions Within each such "coupling" entry, there are four subsections:
    1. A section describing the coupler used, and the coupling frequency for that particular input.
    2. A section describing how (potentially multiple) inputs were gathered from other components to feed into the one you are describing. (Note we are only interested in coupling to components at the "realm" level.)
    3. A section describing how inputs were gathered from (potentially multiple) files (described using the file tab) to feed into the one you are describing,and
    4. A section describing how the inputs were manipulated to produce the final input.
    It is likely that you'll only fill out section (2) or section (3) for any one coupling.
  • Initial conditions Within each such coupling entry, we ask you simply to bind the input to between one and many variables from the file/variable combinations you have entered using the file tab.
  • Ancillary Files Within each such coupling entry, we ask you simply to bind the input to one file (and one file only, if you want multiple files, please use multiple input requirements to distinguish them.

The level of detail to input is up to you, but we will be asking you about how your simulation conformed to the experimental requirements on the simulation "conformance" pages. In many cases that will be asking you to link some response of yours to both a particular requirement and a particular input, so if you don't do it now, you might have to come back later :-) (and obviously the scientific interpretation of the simulation results is significantly enhanced if one knows how the various model components are coupled to each other and to external files. Note also that the simulation description will not validate as complete unless all declared input requirements are coupled somehow to boundary conditions.
